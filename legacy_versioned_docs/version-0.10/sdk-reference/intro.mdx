# Introduction

[LOC SDK](https://www.npmjs.com/package/@fstnetwork/loc-logic-sdk) is a downloadable NPM package that defines the interfaces of data context as well as agents - a series of features provided by LOC's logic runtime that can read and/or write internal and external storages.

The [data context](/legacy/0.10/sdk-reference/context-and-task) contains the following features:

-   [Task](/legacy/0.10/sdk-reference/context-and-task#task)
-   [Payload](/legacy/0.10/sdk-reference/payload)

The following agents can be imported:

-   [Session Storage](/legacy/0.10/sdk-reference/session)
-   [Local Storage](/legacy/0.10/sdk-reference/local)
-   [Event Store](/legacy/0.10/sdk-reference/eventstore)
-   [Result](/legacy/0.10/sdk-reference/result)
-   [Logging](/legacy/0.10/sdk-reference/logging)
-   [Local Variable](/legacy/0.10/sdk-reference/logic-var)
-   [HTTP](/legacy/0.10/sdk-reference/http)
-   [Database](/legacy/0.10/sdk-reference/database)
-   [File Storage](/legacy/0.10/sdk-reference/filestorage)
-   [Mail](/legacy/0.10/sdk-reference/mail)

:::info
This section is currently based on SDK `v0.9.0-1` (which is used for LOC Core `v0.10.2` and Studio `v1.5.1`).

Supported logic code languages/version:

-   JavaScript `ES6/ES2015` and above
-   TypeScript `3.7.0` and above

You do not need to install SDK in Studio. For installing SDK in CLI workspaces, see [Install SDK and Dependencies](/legacy/0.10/cli-handbook/setup#install-sdk-and-dependencies).

Some of the agents require **agent configuration**. See [tutorial](/legacy/0.10/tutorials/http-agent-configuration) for details.
:::

## How to Import Agents

From `v0.7.0` all logic have to _explicitly_ import agents and related classes from LOC SDK using **ECMAScript 6** imports. Below is a short demostration:

:::tip
If you are using [LOC Studio](/legacy/0.10/category/studio-guide), the imports will be automatically included in the code templates.
:::

<Tabs>
  <TabItem value="js" label="JavaScript" default>

```javascript title="Logic" showLineNumbers
// highlight-start
import {
    LoggingAgent,
    SessionStorageAgent,
    EventAgent,
} from "@fstnetwork/loc-logic-sdk";
// highlight-end

export async function run(ctx) {
    // ...
}
```

  </TabItem>
  <TabItem value="ts" label="TypeScript">

```typescript title="Logic" showLineNumbers
// highlight-start
import {
    GenericContext,
    RailwayError,
    LoggingAgent,
    SessionStorageAgent,
    EventAgent,
    Event,
} from "@fstnetwork/loc-logic-sdk";
// highlight-end

export async function run(ctx: GenericContext) {
    // ...
}
```

  </TabItem>
</Tabs>

## Agent Configuration

Four of the agents - **HTTP**, **database**, **file storage** and **mail** - requires **agent configuration** to connect external data sources. You can set them either via [Studio](/legacy/0.10/studio-guide/agent-configuration).
